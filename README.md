<h2><a id="user-content-initialize-sdk" class="anchor" href="#initialize-sdk" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Initialize SDK</h2>
<p>Ensure that you have gone through <a href="https://console.ng.bluemix.net/docs/services/mobilepush/push_step_1.html#push_step_1">Configuring credentials for a notification provider</a> to setup the FCM project and obtain your credentials.</p>
<h3><a id="user-content-include-client-push-sdk-with-gradle" class="anchor" href="#include-client-push-sdk-with-gradle" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Include client Push SDK with Gradle</h3>
<p>Configure the Module level <code>build.gradle</code> and Project level <code>build.gradle</code> files.</p>
<ol>
<li>
<p>Add Bluemix Push Notifications Android SDK dependency to your Project level <code>build.gradle</code> file.</p>
<pre><code>dependencies {
	........
	compile group: 'com.ibm.mobilefirstplatform.clientsdk.android',
     	name: 'push',
    	 version: '3.+',
    	 ext: 'aar',
     	transitive: true
	.......
}
</code></pre>
</li>
<li>
<p>Add the following dependencies to your Module level <code>build.gradle</code> file.</p>
<pre><code> dependencies {
 	 classpath 'com.android.tools.build:gradle:2.2.3'
  	classpath 'com.google.gms:google-services:3.0.0'
}
</code></pre>
</li>
<li>
<p>Add the <code>Google Play services</code> dependency to your Module level <code>build.gradle</code> file at the end, after the <code>dependencies{.....}</code>:</p>
<pre><code>apply plugin: 'com.google.gms.google-services'
</code></pre>
</li>
<li>
<p>Configure the <code>AndroidManifest.xml</code> file. Refer the <a href="https://github.com/ibm-bluemix-mobile-services/bms-samples-android-hellopush/blob/master/helloPush/app/src/main/AndroidManifest.xml">example here</a>. Add the following permissions inside application's <code>AndroidManifest.xml</code> file.</p>
<pre><code>&lt;uses-permission android:name="android.permission.INTERNET"/&gt;
&lt;uses-permission android:name="android.permission.GET_ACCOUNTS" /&gt;
&lt;uses-permission android:name="android.permission.USE_CREDENTIALS" /&gt;
&lt;uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" /&gt;
&lt;uses-permission android:name="android.permission.ACCESS_WIFI_STATE"/&gt;
</code></pre>
</li>
<li>
<p>Add the notification intent settings for the activity. This setting starts the application when the user clicks the received notification from the notification area.</p>
<pre><code> &lt;intent-filter&gt;
 &lt;action android:name="Your_Android_Package_Name.IBMPushNotification"/&gt;
 &lt;category  android:name="android.intent.category.DEFAULT"/&gt;
 	&lt;/intent-filter&gt;
</code></pre>
<blockquote>
<p><strong>Note</strong>: Replace <code>Your_Android_Package_Name</code> in the previous action with the application package name used in your application.</p>
</blockquote>
</li>
<li>
<p>Update the <code>Firebase Cloud Messaging (FCM)</code> or <code>Google Cloud Messaging (GCM)</code> intent service and intent filters for the <code>RECEIVE</code> and <code>REGISTRATION</code> event notifications:</p>
<pre><code>&lt;service android:name="com.ibm.mobilefirstplatform.clientsdk.android.push.api.MFPPushIntentService"
 android:exported="true" &gt;
 &lt;intent-filter&gt;
     &lt;action android:name="com.google.firebase.MESSAGING_EVENT" /&gt;
 &lt;/intent-filter&gt;
&lt;/service&gt;
&lt;service
android:name="com.ibm.mobilefirstplatform.clientsdk.android.push.api.MFPPush"android:exported="true" &gt;
&lt;intent-filter&gt;
   &lt;action android:name="com.google.firebase.INSTANCE_ID_EVENT" /&gt;
&lt;/intent-filter&gt;
&lt;/service&gt;
</code></pre>
</li>
<li>
<p>Push Notifications service supports retrieval of individual notifications from the notification tray. For notifications accessed from the notification tray, you are provided with a handle only to the notification that is being clicked. All notifications are displayed when the application is opened normally. Update your <code>AndroidManifest.xml</code> file with the following snippet to use this functionality:</p>
<pre><code>&lt;activity android:name="com.ibm.mobilefirstplatform.clientsdk.android.push.api.MFPPushNotificationHandler"
   android:theme="@android:style/Theme.NoDisplay"/&gt;
</code></pre>
</li>
<li>
<p>Add the <code>google-services.json</code> in Android application module root directory.  For more information on how to add this file, see <a href="https://console.ng.bluemix.net/docs/services/mobilepush/push_step_3.html#push_step_3_Android">Setup the Push client SDK on FCM</a>.</p>
</li>
</ol>

<h2><a id="user-content-monitoring" class="anchor" href="#monitoring" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Monitoring</h2>
<p>To monitor the current status of the notification within the application, you can implement the <code>com.ibm.mobilefirstplatform.clientsdk.android.push.api.MFPPushNotificationStatusListener</code> interface and define the method <code>onStatusChange(String messageId, MFPPushNotificationStatus status)</code>.</p>
<p>The <code>messageId</code> is the identifier of the message sent from the server. <code>MFPPushNotificationStatus</code> defines the status of the notifications as values,</p>
<ul>
<li>
<p><code>RECEIVED</code> - App has received the notification.</p>
</li>
<li>
<p><code>QUEUED</code> - App queues the notification for invoking the notification listener.</p>
</li>
<li>
<p><code>OPENED</code> - User opens the notification by clicking the notification in the tray or by launching it from app icon or when the app is in foreground.</p>
</li>
<li>
<p><code>DISMISSED</code> - User clears/dismisses the notification in the tray.</p>
</li>
</ul>
<p>You need to register the <code>com.ibm.mobilefirstplatform.clientsdk.android.push.api.MFPPushNotificationStatusListener</code> class with <code>MFPPush</code>.</p>
<pre><code>	push.setNotificationStatusListener(new MFPPushNotificationStatusListener() {
	@Override
	   public void onStatusChange(String messageId, MFPPushNotificationStatus status) {
       // Handle status change
	}
	});
</code></pre>
<h3><a id="user-content-listening-to-the-dismissed-status" class="anchor" href="#listening-to-the-dismissed-status" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Listening to the DISMISSED status</h3>
<p>You can choose to listen to the DISMISSED status on either of the following conditions:</p>
<ul>
<li>When the app is <code>active (running in foreground or background)</code>. Add the snippet to your <code>AndroidManifest.xml</code> file:</li>
</ul>
<pre><code>		&lt;receiver android:name="com.ibm.mobilefirstplatform.clientsdk.android.push.api.MFPPushNotificationDismissHandler"&gt;
    	  &lt;intent-filter&gt;
        &lt;action android:name="Your_Android_Package_Name.Cancel_IBMPushNotification"/&gt;
    	  &lt;/intent-filter&gt;
   	&lt;/receiver&gt;
</code></pre>
<ul>
<li>When the app is both <code>active (running in foreground or background)</code> and <code>not running (closed)</code>. Extend the <code>com.ibm.mobilefirstplatform.clientsdk.android.push.api.MFPPushNotificationDismissHandler</code> broadcast receiver and override the method <code>onReceive()</code>, where the <code>MFPPushNotificationStatusListener</code> should be registered before calling method <code>onReceive()</code> of the base class:</li>
</ul>
<pre><code>		public class MyDismissHandler extends MFPPushNotificationDismissHandler {
		@Override
		public void onReceive(Context context, Intent intent) {
		MFPPush.getInstance().setNotificationStatusListener(new MFPPushNotificationStatusListener() {
		@Override
		public void onStatusChange(String messageId, MFPPushNotificationStatus status) {
		// Handle status change
       	}
     	});
     	super.onReceive(context, intent);
		}
			}
</code></pre>
<p>Add the following snippet to you <code>AndroidManifest.xml</code> file:</p>
<pre><code>	&lt;receiver android:name="Your_Android_Package_Name.Your_Handler"&gt;
	&lt;intent-filter&gt;
	    &lt;action android:name="Your_Android_Package_Name.Cancel_IBMPushNotification"/&gt;
	&lt;/intent-filter&gt;
	&lt;/receiver&gt;
</code></pre>
